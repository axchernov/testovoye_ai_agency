# Telegram Lead Bot

Telegram-бот для предварительной квалификации B2B-заявок на AI-автоматизацию. Бот получает согласие на обработку персональных данных, последовательно задаёт 13 вопросов и отправляет заполненную заявку в личный Telegram владельца.

## Требования

- Node.js 20 или новее;
- Telegram-аккаунт администратора;
- бот, созданный через `@BotFather`.

## Установка

Перейдите в каталог самостоятельного проекта и установите зависимости:

```powershell
cd telegram-lead-bot
npm install
```

## Создание бота и получение BOT_TOKEN

1. Откройте в Telegram официальный аккаунт `@BotFather`.
2. Отправьте команду `/newbot`.
3. Задайте имя и username бота по подсказкам BotFather.
4. Скопируйте полученный API token. Не публикуйте его и не добавляйте `.env` в Git.

## Получение личного ADMIN_CHAT_ID

Самый простой способ — открыть в Telegram `@userinfobot` и нажать Start. Бот покажет числовой id вашего аккаунта.

Можно получить id и через собственного бота:

1. Создайте `.env` сначала только с `BOT_TOKEN`.
2. Откройте своего бота в Telegram и нажмите Start или отправьте `/start`.
3. Пока локальный бот не запущен, откройте в браузере адрес `https://api.telegram.org/bot<BOT_TOKEN>/getUpdates`.
4. Найдите в ответе `message.chat.id` — это ваш личный `ADMIN_CHAT_ID`.

Администратор обязательно должен первым открыть собственного бота и выполнить `/start`. Telegram не позволяет боту первым начинать личную переписку.

## Настройка .env

Скопируйте пример:

```powershell
Copy-Item .env.example .env
```

Заполните `.env` реальными значениями:

```dotenv
BOT_TOKEN=1234567890:your_real_bot_token
ADMIN_CHAT_ID=123456789
```

`BOT_TOKEN` обязателен. Если `ADMIN_CHAT_ID` отсутствует или некорректен, бот продолжит работать и выведет понятную ошибку в консоль, но не сможет доставить финальную заявку.

## Запуск

Режим разработки с автоматическим перезапуском:

```powershell
npm run dev
```

Production-сборка и запуск:

```powershell
npm run build
npm start
```

Бот использует long polling, поэтому отдельный сервер webhook не нужен.

## Автоматические тесты

```powershell
npm test
```

Тесты проверяют порядок вопросов, кнопочные варианты, email и телефон, состояние анкеты, формат заявки, согласие и полный путь доставки администратору.

## Ручная проверка

1. Убедитесь, что администратор выполнил `/start` в личном чате с ботом.
2. Запустите локально `npm run dev`.
3. От другого Telegram-аккаунта выполните `/start`.
4. Проверьте, что без кнопки «Согласен» опрос не начинается.
5. Нажмите «Не согласен» и убедитесь, что сценарий завершается.
6. Снова выполните `/start`, нажмите «Согласен» и пройдите вопросы по порядку.
7. На вопросе с кнопками отправьте обычный текст — бот должен попросить выбрать вариант кнопкой.
8. Введите некорректный телефон и email — бот должен оставить вас на текущем шаге.
9. Выполните `/reset` во время анкеты и убедитесь, что прогресс удалён.
10. Пройдите анкету полностью и проверьте личный чат администратора: туда должно прийти сообщение `🆕 Новая заявка на AI-автоматизацию` со всеми ответами и Telegram-данными клиента.

Состояние хранится только в памяти через `Map` и очищается после успешной отправки. При перезапуске процесса незавершённые анкеты теряются — это ожидаемое поведение проекта без базы данных.
